Getting Started

Where JavaScript Lives

<!-- In your HTML file -->

<script>
  // Your code here
</script>

JS runs in the browser

Your JavaScript lives in a <script> tag, same way CSS lived in a style tag.

The browser reads it top to bottom. We'll watch it run using DevTools.

Core Concept

Variables: let

A variable is a labeled box for a value.

let score = 5;
let playerName = "Sam";

Read it right to left: take the value, put it in a box, label the box.

Debugging

console.log()

Your window into the code — see what's happening.

console.log(score);
// → shows in DevTools console

Whenever something isn't working, your first move is: log it and look.

Reading Input

.value

let typed = input.value;

.value = the current contents of an input

Reaches into an input box and hands you whatever's typed there, as text.

Writing Output

.innerHTML

output.innerHTML = "Hello!";

.innerHTML = the content inside an element

.value reads from an input.
.innerHTML writes into an element on the page.

Data Types

Strings: Text in Quotes

"hello"
'world'

Text values are called strings.

Single or double quotes — your choice, just be consistent.

Combining Text

Glue Strings: +

"Hello, " + name + "!"

The + joins strings end to end.

Watch the spaces — you have to put them inside the quotes yourself.

This works, but it gets fiddly fast. There's a cleaner way...

Upgrade

Template Literals

`Hello, ${name}!`

Backticks + ${ } to drop variables in

Same sentence as the last slide, way easier to read.

Watch Out

Strings vs. Numbers

"7"

is text

≠
7

is a number

Inputs always give you text — even digits.

The Gotcha
"7" + "7" = ?
"77"

Because they're strings, + glues them instead of adding.
The computer did exactly what we told it.

The Fix

Number() to the Rescue

Number("7") + Number("7")
// → 14

Convert text → number before doing math.

Rule of thumb: if you're doing math on something from an input,
wrap it in Number() first.
